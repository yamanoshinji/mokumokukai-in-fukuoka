# GitHub Copilot 入門

## プレミアムリクエスト

- **月間上限：300件**（Copilot Pro / Business プランの場合）
- モデルごとに消費倍率が異なる（0倍〜最大30倍）
  - 消費0倍：GPT-4.1、GPT-4o、GPT-5 mini（有料プランでは無制限に使用可能）
  - 消費1倍：Claude Sonnet 4.6、Gemini 2.5 Pro など標準モデル
  - 消費3倍：Claude Opus 4.5 / 4.6
- 毎月1日にリセット（未使用分は繰り越し不可）
- 使い切っても消費0倍のモデルは引き続き利用可能
- VSCode 右下の Copilot アイコンから現在の使用状況（%）を確認できる

---

## チャットモード

VSCode の Copilot Chat には **Agent / Ask / Plan** の3モードがある。

| モード | 概要 |
| --- | --- |
| **Agent** | ファイル操作・コマンド実行も含む自律的なタスク処理。基本はこれを使う |
| **Ask** | 質問・調査・説明など読み取り専用の対話向け |
| **Plan** | ゼロベースでタスク計画を立てる際に有効 |

> **注意：** Agent モードはファイル変更やコマンド実行に承認が必要。Auto-approve（自動承認）も設定可能だが、意図しない操作のリスクがある。

---

## モデル選択

- **基本は「Auto」でよい。** Auto 選択時はプレミアムリクエスト消費が10%割引になる（有料プランのみ）
- ソース全体の深い推論や高度な分析が必要な場合のみ **Claude Opus 4.5 / 4.6**（消費3倍）を選択する
- 通常のコーディング支援には Claude Sonnet 系や GPT-4.1 で十分

---

## コンテキスト（トークン）管理

- 同一セッションで会話を続けると、モデルが過去のやり取りを忘れたり回答精度が落ちることがある
- VSCode 右下の丸アイコンで現在のコンテキスト使用状況を確認できる
- 適切なタイミングでセッションを分けることを推奨
- 内容を引き継ぎたい場合は、やり取りをファイルに書き出してから新セッションで読み込む
