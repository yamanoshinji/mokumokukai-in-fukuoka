## 2. 今後のネタ
- 概要
  - AIエージェントの構築
  - チャット上で修正内容を伝えると、AIが必要な情報をプロンプトとして整理し、影響範囲や修正箇所を特定します。さらに、クラス図・シーケンス図・ER図・CRUD図などの設計ドキュメントも自動生成し、既存の設計書と比較しながら修正の妥当性を検証できる仕組みを目指す
- 目的
  - 既存システムのカスタマイズ、運用保守業務の効率化
  - 俗人化防止
- GitHub Copilotの下位互換にならないために
  - マルチモーダルな設計提案
    - コードだけでなく、シーケンス図、ER図、CRUD図を同時に生成・提示
  - 設計書ドリブンの整合性検証
    - 「仕様書Aにはこう書いてあるが、今回の修正はそれに違反しないか？」をRAGで検索してチェック
  - 影響範囲の事前可視化
    - 修正によって影響を受ける他モジュールを特定し、実装前にリスクを提示
