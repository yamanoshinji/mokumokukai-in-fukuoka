## 1. まずは「プロジェクトの司令塔」を作る

Copilotを導入したら、まず最初に **プロジェクト専用の指示書（`.github/copilot-instructions.md`）** を作成しましょう

### 導入プロンプトの実行

添付の初期設定プロンプトをチャットに投げてください
公式でおススメされているプロンプトの末尾に「ただし、日本語で出力すること」を加えたものです

### 何が起きるか

Copilotがリポジトリ全体をスキャンし、「このプロジェクトの技術スタック、ディレクトリ構成、ビルド方法、コーディング規約」をまとめた指示書を自動生成します

### 効果

次回以降、Copilotはこの指示書を「絶対のルール」として参照します
これにより、プロジェクトの文脈から外れた的外れな回答が激減します

## 2. 「Plan → Agent」の黄金サイクルで開発する

VSCode上のCopilotには **Ask（質問）**, **Plan（計画）**, **Agent（実行）** の3モードがあります
(昔と変わっていて、Editがなくなり、Planが追加されています)

### 1. Planモードで「設計図」を作る

「〇〇機能を実装して」と頼むと、まずどのファイルをどう書き換えるかの計画を立ててくれます
ここで人間が「その方針でOK」とレビューするのが重要です

### 2. Agentモードで「並行処理」させる

計画が決まったらAgentに切り替えます
AIが複数のファイルを一気に書き換え、ターミナルでビルドし、エラーが出れば自分で修正して…という一連の作業を代行します

### 3. 確認と調整

AIが誤解した箇所があれば、チャットで指摘するだけでなく、その原因を指示書（`instructions.md`）にフィードバックしましょう
これが「AIを育てる」ということです

## 3. 寝ている間に仕事を進める「GitHub Coding Agent」

VSCodeを閉じた後も、Copilotに働いてもらいましょう
※そのためには、CI/CDをしっかり組んでおくことが重要です

### Issueからの自動実装

GitHub上のIssueにCopilotをアサイン（Assign）すると、クラウド上で解決策を考え、PR（プルリクエスト）を自動作成してくれます

### CI/CDエラーの自動修正

テストが落ちた際、Copilotに「エラーを直してPRを更新して」と指示すれば、バックグラウンドで修正作業が進みます

### 注意

クラウド実行は非常に便利ですが、複雑なロジックは精度が落ちることもあるため、翌朝しっかり人間がレビューしてください

## まとめ：AIは「指示を待つツール」から「共に働くパートナー」へ

GitHub Copilotを使いこなす鍵は、 **「AIが迷わないためのコンテキスト（文脈）を整えること」** に尽きます。

1. **AIを育てる：** プロジェクト固有の知見を `copilot-instructions.md` に蓄積する

2. **モードを使い分ける：** 計画（Plan）と実行（Agent）を分離して管理する

3. **クラウドを活用する：** 非同期作業で自分の時間を最大化する

「AIに全部やらせる」のではなく、「AIが最高の結果を出せるようにマネジメントする」感覚をぜひ楽しんでください
