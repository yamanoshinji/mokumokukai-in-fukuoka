# 📚 今までのもくもく会で学んだGit機能（及びコマンド）一覧

## 🌱 初心者向け（日常的に使う基本コマンド）

- git init：現在のディレクトリに空の .git ディレクトリ（リポジトリの心臓部）を作成する。
- git clone：リモートからリポジトリを複製する
- git branch：ブランチ（特定のコミットを指すポインタ）の一覧表示や作成・削除を行う。
- git add：ワークツリーの変更をステージ（インデックス）に登録する
- git commit：ステージの情報をローカルリポジトリに歴史（Commitオブジェクト）として記録する
- git push：ローカルの歴史をリモートリポジトリに送信する
- git fetch：リモートの最新情報（origin/mainなど）をPC内の隠しコピーとして取得する
- git merge：取得した歴史（origin/mainなど）や他のブランチを現在のブランチに合体させる
- git pull：fetchとmergeを連続で行う
- git checkout / git switch：HEADポインタを移動させ、ワークツリーを指定したコミットやブランチの状態に書き換える
- git diff：現在のHEAD（コミット）と指定したコミットなどの差分を確認する

## 🛠️ 中級者向け（歴史の操作や部分的な反映）

- git reset (--soft, --mixed, --hard)：HEADポインタを過去へ強制移動させ、歴史を無かったことにする
- git rebase (および -i)：ブランチの根元を付け替えたり、過去のコミット履歴を綺麗に書き換える
- git revert：過去の特定のコミットの変更を打ち消す「新しいコミット」を作成する
- git cherry-pick：他のブランチから特定のコミットの変更内容だけをつまみ食いして現在のブランチにコピーする
- git stash (および pop, -u)：ワークツリーとステージの未完了な変更を一時的に退避（隠す）させる
- git remote：リモートリポジトリの参照設定を行う（set-headなどでorigin/HEADを更新）

## 🔍 まだ勉強会で触っていないGit機能（及びコマンド）一覧

## 🌱 初心者向け（構造理解のために触っておきたい基本）

- git status：現在のワークツリーとステージの状態、HEADがどこにいるかを確認する（一番使う！）
- git log：HEADから辿れるコミットの歴史（グラフ）を確認する。

## 🛠️ 中級者向け（調査・復旧・高度な操作）

- git restore：ワークツリーやステージの変更を取り消す（昔はcheckoutが担っていた役割を分離したもの）。
- git reflog：HEADポインタの移動履歴を全て記録しているログ。reset --hardで消してしまったコミットを復旧させるための最後の命綱！
- git tag：特定のコミットに対して、動かない不変のラベル（v1.0.0など）を付与する。
- git blame：ファイルの行ごとに「誰が、いつ、どのコミットで書いたか」を特定する（バグ調査で大活躍）。
- git bisect：二分探索を用いて、バグが混入したコミットを自動的に特定する（歴史が長いプロジェクトで超便利）。
