# もくもく会記録を Backlog に投稿するガイド

`01_Feedback/` 配下の勉強会記録（日付フォルダ）を、Backlog の課題として投稿するための手順とスクリプトです。
Markdown を Backlog 記法へ自動変換し、画像はそのまま添付ファイルとして紐付けます。

## 前提

- Node.js **18 以上**（`fetch` / `FormData` を使用。`node -v` で確認）
- Backlog の **個人 API キー**（Backlog → 個人設定 → API で発行）

> ⚠️ **API キーは絶対にコミットしないこと。** 必ず環境変数 `BK_KEY` で渡します。

## 投稿手順

### 1. API キーを環境変数にセット

```bash
# Git Bash / macOS / Linux
export BK_KEY="＜あなたの Backlog API キー＞"
```

```powershell
# PowerShell
$env:BK_KEY="＜あなたの Backlog API キー＞"
```

### 2. まず変換結果を確認（投稿はしない）

```bash
node tools/backlog/post_backlog.js dry 20260415   # 指定日付だけ
node tools/backlog/post_backlog.js dry             # 全日付
```

件名・添付画像・本文（Backlog 記法）が表示されます。問題なければ次へ。

### 3. 投稿する

```bash
node tools/backlog/post_backlog.js post 20260415   # 新しく追加した日付だけ
node tools/backlog/post_backlog.js post all        # 全日付（既存も再投稿されるので注意）
```

> `post all` は対象フォルダを**すべて新規課題として投稿**します。既に投稿済みの日付は重複するので、
> 通常は **新しく追加した日付だけを指定**して投稿してください。

### 4. （任意）投稿と同時に 親ぶら下げ / 担当者 / ステータス を設定

```bash
# 例: もくもく会(157206233) にぶら下げ、担当者=山野(1868462)、ステータス=チャレンジ中(419120)
PARENT_ID=157206233 ASSIGNEE_ID=1868462 STATUS_ID=419120 \
  node tools/backlog/post_backlog.js post 20260415
```

これだけで「作成 → 親にぶら下げ → 担当者設定 → ステータス変更」まで一括で済みます。

## 新しい記録の書き方（テンプレート）

`01_Feedback/YYYYMMDD/YYYYMMDD.md` を作り、画像は同じフォルダに置きます。
フォルダ名（8 桁の日付）が課題の日付になり、`内容：` の行が件名末尾に使われます。

```markdown
# 勉強会記録：もくもく会in福岡支社

## 1. 概要
- 日付：YYYY/MM/DD
- 内容：（ここが課題の件名末尾になります）
- 参考記事/動画：
  - [タイトル](https://example.com/...)

## 2. 学んだこと（議論になったところ中心）
- 箇条書き
  - ネスト（2スペース＝1段）
![説明](image.png)

| 見出しA | 見出しB |
| --- | --- |
| 値1 | 値2 |
```

- 課題の件名は自動で `もくもく会 YYYY/MM/DD <内容>` になります。
- 画像（`*.png` / `*.jpg` / `*.gif`）は自動で課題に添付されます。
- メインの `YYYYMMDD.md` 以外の `.md` は「補足資料」として本文末尾に追記されます。

## Markdown → Backlog 記法の変換ルール

| Markdown | Backlog 記法 |
|---|---|
| `# 見出し` / `## 見出し` | `* 見出し` / `** 見出し` |
| `- 項目` / `  - 子項目` | `- 項目` / `-- 子項目` |
| `1. 項目` | `+ 項目` |
| `**太字**` | `''太字''` |
| `[ラベル](https://…)` | `[[ラベル:https://…]]` |
| `[ラベル](相対パス)` | `ラベル（相対パス）`（リンク化しない） |
| ` ```code``` ` | `{code} … {/code}` |
| 表（`\| … \|`） | ヘッダ行末に `h` を付与・区切り行は除去 |
| `![alt](image.png)` | `（添付画像: image.png — alt）`（画像本体は課題に添付） |

> Backlog 上のリンクで解決できないリポジトリ内相対パスは、リンク化せずテキスト表記にしています。
